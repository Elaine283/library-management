🔒 ACID 交易特性展示
A - 原子性
借書操作包含：
1. 檢查庫存
2. 更新書籍狀態
3. 插入借閱紀錄
要嘛全成功，要嘛全回滾
C - 一致性
業務層驗證 + 手動交易控制， 確保書籍狀態和借閱紀錄 時刻保持同步， 永遠無髒資料
I - 隔離性
每個 Connection 獨立管理， 互不干擾， 防止髒讀與衝突
D - 持久性
commit() 後即同步到 AWS RDS MySQL， 資料永久保存
⚡ 核心代碼流程（手動交易控制）
// 1. 取得原生資料庫連線
Connection conn = databaseConfig.getConnection();

try {
    // 2. 【關鍵】關閉自動提交，開啟手動交易控制
    conn.setAutoCommit(false);

    // 3. 業務邏輯：檢查 → 更新 → 插入
    Book book = bookDAO.getBookById(conn, bookId);
    bookDAO.updateBookStatus(conn, bookId, "BORROWED");
    borrowRecordDAO.insertRecord(conn, bookId, userId);

    // 4. 通過考驗：全面提交交易
    conn.commit();

} catch (Exception e) {
    // 5. 異常回滾：一旦失敗，全面無條件回滾
    conn.rollback();
    throw e;
} finally {
    // 6. 確保釋放連線
    conn.close();
}
